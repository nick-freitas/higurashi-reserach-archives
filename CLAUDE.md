# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Purpose

This is Higurashi Archives - a community-maintained bilingual (Japanese/English) translation resource for the visual novel "Higurashi: When They Cry" by 07th Expansion. It provides dialogue and text data used by the 07th-Mod fan translation project.

## Higurashi Narrative Structure

Understanding Higurashi's unique structure is essential when working with this data:

### Arc-Based Storytelling
The story consists of multiple **arcs** (episodes), each identified by filename prefix. The same core cast appears across arcs, but **arcs do not necessarily share continuity**:

- **Connected arcs** - Some occur in the same timeline with events carrying over
- **"What if" arcs** - Alternative scenarios where different choices lead to different outcomes
- **Answer arcs** - Later arcs that reveal truths about earlier "question" arcs

### Visual Novel Branching
Reader choices create branching paths leading to good/bad endings, different character fates, and revelations that recontextualize earlier events.

### Implications
A character may behave very differently between arcs—an antagonist in one arc may be sympathetic in another, deaths or betrayals in one arc may not occur in others. When analyzing content, always consider which arc it comes from and whether observations are arc-specific or consistent across the narrative.

## Repository Structure

This is a web application (React + Express) with bilingual game text data:

- **src/** - React frontend (components, utils, tests)
- **server/** - Express API server (indexer, search, arc browser)
- **game_text/** - JSON dialogue/text files organized by filename prefix:
  - **tips_###.json** (188 files) - Game tips/trivia content
  - **fragments_X#.json** (102 files) - Character perspective fragments
  - **common_*.json** (43 files) - Shared common storyline
  - **miotsukushi_*.json** (47 files) - Miotsukushi chapter
  - **[chapter]_##.json** - Individual story arc chapters
  - **chapternames.json** - Master chapter/route mapping metadata
- **py_scripts/** - Python translation/scoring scripts
- **scripts/** - Python utility scripts
- **characters/** - Character profile files generated by the CharacterAnalyzer agent
- **data/** - Runtime data (annotations)
- **docs/** - Design specs and plans
- **tests/** - Test files

## JSON Content Format

Each file contains an array of objects with bilingual entries:

```json
[
  {
    "type": "SAVEINFO",
    "category": 0,
    "JPN": "Japanese text",
    "ENG": "English translation"
  },
  {
    "type": "MSGSET",
    "MessageID": 12345,
    "NamesJPN": "Character name (JP)",
    "NamesENG": "Character name (ENG)",
    "PreTextTags": "@vS01/01/120100156.",
    "TextJPN": "Japanese dialogue",
    "TextENG": "English translation"
  }
]
```

**Entry Types:**
- **SAVEINFO**: Save point/checkpoint labels
- **MSGSET**: Character dialogue and narration

**Key Fields:**
- `MessageID`: Unique message identifier
- `PreTextTags`: Voice file references
- `NamesJPN`/`NamesENG`: Character names in respective languages
- `TextJPN`/`TextENG`: Content in respective languages

## Custom Agents

### CharacterAnalyzer

Defined in `.claude/agents/CharacterAnalyzer.md`. Analyzes dialogue files to build character profiles with awareness of Higurashi's disjointed continuity.

**Usage:** `Analyze [character name]` (e.g., "Analyze Rena")

**What it does:**
- Searches JSON files for character appearances in `NamesENG`/`NamesJPN` fields
- Extracts dialogue, speech patterns, and relationships
- Distinguishes arc-specific traits vs consistent traits across arcs
- Tracks how characters differ between "what if" scenarios
- Creates or updates character profile in `characters/[name].md`
